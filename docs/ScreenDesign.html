Screen Design — 개요

본 문서는 화면 기능 명세서(Screen Spec)를 기반으로, 각 화면의 레이아웃 구조 · UI 요소 배치 · 인터랙션 흐름 · 상태별 화면 변화를 시각적으로 명세합니다. 키오스크(K) 화면과 관리자 앱·웹(A) 화면으로 구분되며, 와이어프레임 다이어그램과 설계 주석을 함께 제공합니다.

문서 목적 및 활용 방법
  • 설계 기준 문서 — 개발팀이 실제 구현 시 참조하는 UI 레이아웃 및 인터랙션 기준입니다. 화면 기능 명세서(Screen Spec)와 함께 사용합니다.
  • 와이어프레임 기반 — 각 화면은 블록형 와이어프레임으로 영역 구조를 시각화하며, 실제 디자인 시안 제작 전 구조 합의용으로 활용합니다.
  • 레이아웃 영역 주석 — 와이어프레임 각 블록에는 UI 요소 타입(BTN·CAM·INP 등)과 역할 설명이 병기됩니다.
  • 인터랙션 흐름 — 사용자 조작·시스템 처리·타이머 등 트리거별 화면 전환 경로를 플로우 행으로 명세합니다.
  • 상태 설계 — 정상·오류·비활성·로딩 등 각 UI 요소의 상태별 표시 방식을 별도 명세합니다.
  • 협의 필요 항목 — ⚠ 협의 필요 태그로 표시된 항목은 개발 착수 전 양사 합의가 필요합니다.
  • 버전 구분 — ver.1.0은 상세 설계, ver.1.1·1.2는 요약 설계로 제공됩니다. 각 버전은 색상 배지로 구분됩니다.

설계 원칙 및 제약 사항
  • AI 보조 원칙 — AI는 출입 판단 보조 역할만 수행. 최종 판단 권한은 항상 관리자에게 있으며, 이를 UI에서 명확히 표시.
  • 커프 2차 검증 상시 제공 — 모든 측정 결과 화면에서 커프 혈압계 2차 검증 경로를 항시 접근 가능하도록 설계.
  • 개인정보 보호 — 얼굴 인식·혈압 동의 별도 분리. 측정값 비식별 암호화 저장. 통계 화면 집계 데이터만 노출.
  • 면책 문구 고정 — "본 측정값은 의료 진단이 아닙니다" 문구는 키오스크 모든 화면 하단에 항상 고정 표시 (법적 필수).
  • 대체 인증 경로 상시 제공 — 얼굴 인식·RFID·지문 인증 실패 시 이름 직접 입력(K-2-02)으로 언제든 전환 가능.
  • 타임아웃 자동 복귀 — 모든 키오스크 화면은 일정 시간 미조작 시 대기 화면(K-1-01)으로 자동 복귀. 복귀 시간은 화면별 명세 참조.
  • 오프라인 대응 — 네트워크 단절 시에도 측정 기능은 유지. 데이터 로컬 저장 후 복구 시 자동 동기화.
