기능별 피드백 현황
  • 얼굴 인식: 측면 모델 적용 시 오작동 우려로 정면 인식만 유지. 주변 배경/타인 인식을 막기 위한 측정 범위(ROI) 박스 조정 필요.
  • 체온 센서: 측정값이 들어오지 않거나 멈추는 현상 발생. 기존 값 유지 대신 재진입 유도 로직 필요.
  • 디스플레이: 터치감 저하 및 측정 단계 인지 부족. 대기화면(홍보 영상)과 측정화면 간의 명확한 전환 UI 요구됨.
하드웨어 단가 절감 계획
  • 현재 사용 중인 고가의 완제품 대신 모듈(기성품) 단위 구매로 전환 검토.
  • 카메라 모듈 변경을 통해 원가를 50만원대에서 10만원대로 대폭 절감 목표.
  • 추후 코레일 납품을 고려하여 비접촉식 반도체식 음주측정기 통합안 논의 중.
Confirmed Decisions

회의를 통해 최종 확정된 기획 및 정책 사항입니다.

🖥️ UI/UX 및 대기화면 정책
▼
  • 영상 대기화면: 대기 상태에서는 가온병원 등 홍보 영상을 전체 화면으로 송출.
  • 화면 전환: 화면 터치 시 또는 "근무 투입전 적합성 심사" 안내 슬라이드 노출 후 실제 측정 UI로 전환되도록 플로우 확정.
  • 재측정 유도: 체온 측정 실패(데이터 누락) 시, 화면 밖으로 나갔다가 다시 들어오도록 안내 멘트 노출 후 열만 재측정.
⚙️ 측정 로직 (얼굴/체온)
▼
  • 정면 얼굴 인식: 측면 모델 배제. 정면만 인식하며, 뒷사람이 잡히지 않도록 특정 네모 박스(라벨링 영역) 안에 들어올 때만 인식.
  • 커스텀 환경 설정: 현장마다 조명이나 배경이 다르므로 설치 시 최적화된 초기 설정값(표준 범위) 세팅 기능 제공.
⚖️ 법적 고지 및 기타
▼
  • 의료기기 면책 조항: 디지털 의료기기 정식 허가 전이므로, 하단에 반드시 "본 기기는 전문 진단/치료 목적의 의료기기가 아닙니다" 텍스트 상시 노출.
  • 윈도우 라이선스: 원격 관리 지원 및 단속 대비를 위해 Windows 11 Pro 정품 라이선스 도입 확정.
Development Scope

버전별(단기 및 중장기) 개발 목표 범위입니다.

v1.0 단기 목표 (가온병원 납품용)
  • 체온 센서 오류 대응: 값 미수신 시 재측정(이탈 후 재진입) 로직 추가
  • 얼굴 인식 범위 제한: 지정된 ROI 박스 내에서만 얼굴 인식 활성화
  • 홍보 영상 송출 모드: 대기 상태에서 동영상 루프 재생 및 터치 시 UI 전환
  • 법적 문구 삽입: 화면 하단 의료기기 면책 텍스트 영구 노출
  • 설치용 .exe 빌드 안정화: 다수 PC 환경에서 동일 작동 확인
v2.0 중장기 목표 (하드웨어 교체 및 고도화)
  • 저가형 모듈 연동: 기성품 카메라 모듈 구매 후 신규 인터페이스 연동
  • 음주측정기 연동: 반도체식 음주측정기(기성품)를 키오스크 파이프라인에 통합 (스무스한 스크리닝 용도)
  • 관리자 대시보드: 개인별 심박수, 측정 이력 등 월간 리포팅 기능 및 그래프 시각화
  • 커스텀 설문 기능: 구글 폼 형태의 사전 진료/문진표 응답 기능 (측정 완료 후 노출)
